1 Introduction

The need for the spec-gen came from the need for long technical documents to include niceties such as cross-references and a table of contents for the purpose of easy navigation: doing this manually can be a great chore especially when sections are numbered and a section is added, consquently changing the numbering of many others, leading to it being advantageous to do it programmatically.

2 Installing the spec-gen

2.1 Requirements

The following are the minimum requirements: later versions should also work without issue.

  • Python 2.5
  • lxml 2.0
  • html5lib 0.10

2.2 Obtaining a copy

Releases are occasionally made. A link to the latest release can be found from the spec-gen website.

Alternatively, a copy can be obtained from our Mercurial repository: this is where our ongoing development occurs, and allows any revision (and therefore any release) to be downloaded. Our repository is located at http://hg.gsnedders.com/spec-gen/.

2.3 Installation

Normally, installation is done through setuptools, with the following command:

python setup.py install

Or in Unix-like environments:

make install

2.4 Running the test suite

The source distribution and the current development copy (in Mercurial) both contain a test suite. It can be run with the following command:

python runtests.py

Or in Unix-like environments:

make test

Any test failures should be reported at our bug tracker.

3 Processes

The elements listed in the below processes, except where otherwise stated, are the local name of the element in null namespace.

The textContent property is the same as that defined in DOM Level 3 Core on the Node interface.

3.1 Cross-referencing

Cross-referencing has two essential parts: terms that define what something is, and instances of those terms.

Terms are marked-up using the dfn element: the term itself is taken from the title attribute if it is present, otherwise it is taken from the textContent property of the dfn element. By default, the spec-gen will throw a fatal error if a term is defined more than once: this behaviour can be turned off (causing the final definition of the term to be the one that is used) by the --allow-duplicate-terms option.

Instances are marked-up with various elements, depending on the setting of --w3c-compat-elements: if it is disabled (the default), the abbr, code, i, span, and var elements are used for instances; if it is enabled, the abbr, acronym, b, bdo, big, code, del, em, i, ins, kbd, label, legend, q, samp, small, span, strong, sub, sup, tt, var elements are used for instances. Those that are only there in compatibility mode are there because either they shouldn't semantically be used for an instance, or because they aren't present in HTML 5. Similar to terms, the instance is taken from the title attribute if it is present, otherwise it is taken from the textContent property. An instance is only used if it does not have a a, dfn, or datagrid element as either a parent or a child.

Both terms and instances are normalized as follows:

  • Leading and trailing whitespace is stripped,
  • Converted to lowercase (behaviour of this is dependant on the current locale setting of Python),
  • All consecutive whitespace is replaced with a single U+0020 SPACE CHARACTER, and
  • If --w3c-compat-xref-normalization is enabled, all characters apart from U+0020 SPACE CHARACTER, U+002D HYPHEN-MINUS (-), U+0030 DIGIT ZERO to U+0039 DIGIT NINE (0–9), U+0041 LATIN CAPITAL LETTER A to U+005A LATIN CAPITAL LETTER Z (A–Z), and U+0061 LATIN SMALL LETTER A to U+007A LATIN SMALL LETTER Z (a–z) are removed.

If the instance is contained within a span element, the span element is turned into an a element, and a href attribute is added to link it to the term (e.g., <span>foo</span> becomes <a href=#foo>foo</a>) — all other attributes are preserved. Otherwise (when the instance is not contained within a span element), the location of the a element when linking an instance is dependant on the --w3c-compat-xref-a-placement option: if it is disabled (the default), the a element is placed around the element containing the instance (e.g., <i>foo</i> becomes <a href=#foo><i>foo</i></a>); if it is enabled, the a element goes within the element containing the instance and goes around all of its content (e.g., <i>foo</i> becomes <i><a href=#foo>foo</a></i>).
