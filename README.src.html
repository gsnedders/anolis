Introduction

The need for the spec-gen came from the need for long technical documents to include niceties such as cross-references and a table of contents for the purpose of easy navigation: doing this manually can be a great chore especially when sections are numbered and a section is added, consquently changing the numbering of many others, leading to it being advantageous to do it programmatically.

Processes

Cross-Referencing

Cross-referencing has two essential parts: terms that define what something is, and instances of those terms.

Terms are marked-up using the dfn element: the term itself is taken from the title attribute if it is present, or from the textContent property of the dfn element. By default, the spec-gen will throw a fatal error if a term is defined more than once: this behaviour can be turned off (causing the final definition of the term to be the one that is used) by the --allow-duplicate-terms option.

Instances are marked-up with various elements, depending on the setting of --w3c-compat-elements: if it is disabled (the default), the abbr, code, i, span, and var elements are used for instances; if it is enabled, the abbr, acronym, b, bdo, big, code, del, em, i, ins, kbd, label, legend, q, samp, small, span, strong, sub, sup, tt, var elements are used for instances.
